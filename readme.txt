Программа позволяет заказать еду или (роллы или пиццу) по указанному адресу. Также добавлено окно "оплаты".

Программа удовлетворяет следующим требованиям к содержанию проекта:
1)Тема для приложения выбрана студентом самостоятельно
2)Приложение создано в среде Microsoft Visual Studio (версия 2019 года) с нуля и имеет графический интерфейс WPF
3)Приложение имеет многокомпонентную архитектру. Оно содержит запускаемый проект FoodOrder и библиотеку классов Dishes
4)Наличие хранилища данных в виде типизированной коллекции (MainWindow.xaml.cs, строки 27-29). Постоянное хранилище данных представлено в виде двух XML-файлов pizza.xml и rolls.xml
5)Наличие методов для работы с этим хранилищем (GetData() для записи, Delete_Item() для удаления, SetSecondComboBox() для получения информации. Есть и другие методы. Все находятся в MainWindow.xaml.cs
6)Использование интерфейсов (IDish)
7)Использование механизма исключений (MainWindow.xaml.cs, методы GetData(), AddToOrder())
8)Использование свойств, а не полей для хранения данных (классы Rolls и Pizza)
9)Переопределение операций (в классах Rolls и Pizza переопределён метод toString(), унаследованный от Object)
10)Использование LINQ to object: в MainWindow.xaml.cs строках 119-124 конструкции LINQ используются для сортировки объектов в списке в алфавитном порядке их имён (Name)
11)Работа с файлами: данные хранятся в xml-файлах. За их получение отвечает метод GetData(). Чек по итогам заказа формируется в формате .txt и сохраняется в указанном пользователем месте. За это отвечает метод Save_Receipt (строка 238 MainWindow.xaml.cs)
12)Использование стандартных диалогов. В Save_Receipt начиная со строки 241 используется Microsoft.Win32.SaveFileDialog
13)Каждый метод MainWindow.xaml.cs снабжён комментарием, поясняющий, за что метод отвечает. Комментариями снабжены и некоторые другие участки кода.

Так же в код добавлены следующие элементы:

14)Лямбда-выражения(MainWindow.xaml.cs, метод Calc_Delivery строка 217; Rolls.cs строка 49; Pizza.cs строка 43)
15)механизм делегатов(MainWindow.xaml.cs метод Calc_Delivery)
16)блок using, обеспечивающий управление памятью (вызывающий метод Dispose())
17)ещё одно окно(PayWindow), помимо главного, передача информации между окнами.
